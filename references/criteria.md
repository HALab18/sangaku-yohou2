# 登山指数・計算方法の基準表

## 登山指数 A/B/C の判定基準

3時間ブロックごとに以下を判定し、**最悪値**をそのブロックの指数とする。
日別指数は行動時間帯（5〜17時）のブロックの最悪値。
風速・降水の閾値は**予報対象日の月**で夏山／冬山・残雪期モードを自動切替する。

**夏山モード（6月〜10月）**

| 項目 | A（適） | B（注意） | C（不適） |
|---|---|---|---|
| 稜線風速 | < 10 m/s | 10〜15 m/s | ≥ 15 m/s |
| 降水量(3h) | < 1 mm | 1〜5 mm | ≥ 5 mm |
| CAPE(雷指標) | < 500 J/kg | 500〜1000 | ≥ 1000 |

**冬山・残雪期モード（11月〜5月）**

| 項目 | A（適） | B（注意） | C（不適） |
|---|---|---|---|
| 稜線風速 | < 8 m/s | 8〜12 m/s | ≥ 12 m/s |
| 降水量(3h) | < 1 mm | 1〜3 mm | ≥ 3 mm（降雪リスク） |
| CAPE(雷指標) | < 500 J/kg | 500〜1000 | ≥ 1000 |

- **降水確率は判定に使わない**（表には参考値として表示のみ）。判定に使うのは実測降水量(3h)
- 指数の意味: A=登山適 / B=要注意(経験者向き・行程短縮検討) / C=登山不適（表示はA/B/Cのみ、記号は付けない）
- てんきとくらす等の「登山指数」と似た思想だが独自基準。絶対視しないこと。
- モード切替は月のみで判定（山域・標高による個別調整はしない）。
  冬は同じ風速でも危険度が大きく上がるため閾値を8/12m/sに引き下げている。

## 夕方急変の警告フラグ（⚠夕方）

行動時間帯外の 17〜20時 ブロックが C 相当と判定された場合、
日別指数（A/B）は変えずに **「⚠夕方」警告** を付記する。
下山遅れ・テント泊・ご来光待ちで夕方以降も行動する場合の注意喚起。

## 稜線風の算出

Open-Meteo の気圧面風速を山頂標高で線形補間。10m風（地形に沿った地上風）より
気圧面風のほうが稜線上の実勢に近い。

| 気圧面 | 標準高度(m) | 対応する山の例 |
|---|---|---|
| 925hPa | 760 | 筑波山、高尾山 |
| 900hPa | 990 | 泉ヶ岳 |
| 850hPa | 1460 | 丹沢、蔵王、谷川岳 |
| 800hPa | 1950 | 月山、那須、美ヶ原 |
| 700hPa | 3010 | 北アルプス主稜線、八ヶ岳上部 |
| 600hPa | 4200 | 富士山頂 |

## 体感温度

風冷指数（Wind Chill Index, JAG/TI式）で算出する。

```
V_kmh = 稜線風速(m/s) × 3.6
体感温度(℃) = 13.12 + 0.6215×T − 11.37×V_kmh^0.16 + 0.3965×T×V_kmh^0.16
（T = 山頂気温[℃]。風速4.8km/h未満では気温をそのまま採用）
```

従来の慣用則（風速1m/sで−1℃）は強風域（12m/s以上）で体感を過小評価しやすいため置き換えた。
体感温度はA/B/C指数には組み込まず、独立した参考値として扱う。
湿潤時（雨・汗濡れ）はさらに大きく下がるので注意。

## CAPE（対流有効位置エネルギー）の目安

| CAPE (J/kg) | 雷雨の可能性 |
|---|---|
| < 300 | 低い |
| 300〜1000 | 雷雨の可能性あり（夏山午後は特に注意） |
| 1000〜2500 | 雷雨発生しやすい |
| > 2500 | 激しい雷雨・危険 |

CAPEが低くても上空寒気やトリガー次第で雷は起きる。夏山では「13時までに下山開始」が原則。

## 眺望指数 ◎/○/△/✕

山頂標高がかかる雲層（<2000mの山=下層雲/中層雲、≥2000mの山=中層雲）を「山頂のガス」、
山頂より下の層の雲を「雲海の可能性」として判定する。

| 判定 | 条件 |
|---|---|
| ✕ | 3h降水≥1mm / 視程<2km / 山頂レベル雲量≥80% |
| △ | 山頂レベル雲量≥50% または 視程<10km |
| ○ | 山頂レベル雲量<50% |
| ◎ | 山頂レベル雲量≤20% かつ 視程≥20km |
| (雲海) | 山頂より下の雲量≥60% かつ 山頂レベル≤30% → ○/◎に付記 |

- 日別サマリの「眺望(朝)」は4〜8時の最良値（ご来光・朝の展望の目安）
- 雲量バンド（下層<2km/中層2〜7km/上層>7km）は近似。雲頂・雲底高度は数値予報では正確に
  出ないため、稜線ぴったりの雲（滝雲・ガスの湧き）は判定できない
- 低山（<2000m）の谷霧・盆地霧は表現できない
- 上層雲(すじ雲等)は眺望を妨げないため判定に含めない（ご来光の焼けはむしろ絵になる）

## 直近実況（過去3日）の表示

予報とあわせて直近3日のモデル解析値（実況相当）を取得し、予報の前に表示する。
直近の降水・降雪・積雪深の経過を、登山道の状態（ぬかるみ・増水・残雪等）を
利用者自身が見立てるための材料として提供する。
観測所の実測ではないため「モデル解析値」と明記する。
**A/B/C判定には使わない**（表示のみ）。断定的なコンディションのコメントは付けない。

## 積雪の表示

- hourly `snow_depth`（積雪深, m→cm表示）と daily `snowfall_sum`（新雪, cm）を使用
- 表示形式「85cm(+12)」= 積雪深max 85cm・新雪12cm。3時間詳細はブロック内の値
- 取得期間内に積雪・降雪が一度も無い場合は積雪列そのものを省略する（夏山の表を簡潔に保つ）
- A/B/C判定には組み込まない（降雪は降水量として間接的に反映される）

## 横断検索の「日和スコア」（docs/find.html 専用の簡易指標）

「その日に天気の良さそうな山をまず見つけたい」ための逆引きページ（docs/find.html、
生成: scripts/gen_find.py）で使う**簡易スクリーニング指標**。上記 A/B/C の正式判定とは
**別物**であり、CLI/Web 共通の判定ロジックには一切影響しない（規約3の対象外）。

- 目的: エリア内の多数の山を daily 値だけで**バッチ取得**し、相対的な良し悪しで並べる「ふるい」。
  1回の取得は最大50座（負荷抑制のため50座ごとに分割・直列取得）。
- スコア（0〜100・減点方式）の重み順:
  1. **晴天度（最重視）**: 日照率 = `sunshine_duration / daylight_duration` を主指標
     （取れない時のみ `weather_code` で代替）。雨雪雷の天気コードは軽く上乗せ減点。
  2. 降水: `precipitation_probability_max` / `precipitation_sum`。
  3. 風: `wind_speed_10m_max` / `wind_gusts_10m_max`（地表10m値。稜線の目安）。
  4. 雪・寒気: `snowfall_sum` / `temperature_2m_min`。
- **限界（正式判定でない理由）**: daily 値ベースで、A/B/C判定のような**山頂への標高補間
  （700/850hPa風）・時間帯別・CAPE・眺望は行わない**。地表グリッド値のため低山が有利に出やすい。
  あくまで**エリア内の相対比較の入口**であり、実際の登山可否は各山の詳細ページ（正式判定）で確認する。
- 表示: 天気列は「日照率ベース」で出す（daily.weather_code の24hmaxは表示主指標にしない。
  短時間の霧雨が晴主体の日を乗っ取り「曇りなのに日照97%」の矛盾を生むため）。降水リスクは降水列で示す。

## データソースと限界

- 予報: Open-Meteo API（気象庁GSM/MSM・ECMWF・GFS等のベストマッチ合成）
- 予報範囲: 最長16日先（後半は精度が大きく低下、3日先までが実用域）
- 直近実況もモデル解析値であり、観測所・アメダスの実測とは差がある
- 山岳の局地現象（滝雲、局地的突風、雷の正確な位置・時刻）は数値予報では表現不可
- モデル間比較（常時表示）で結果が割れる場合は予報の信頼度が低い
