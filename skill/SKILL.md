# 山岳地点予報スキル (sangaku-yohou)

特定の山の山頂標高に合わせた数値予報を Open-Meteo API から取得し、
登山向けに整形・解説するスキル。実装本体は `{{REPO_PATH}}\` にある
（scripts/mountain_weather.py・references/mountains.csv・references/criteria.md）。

## 手順

1. **山名と日付をパースする。** 「来週末」等の相対表現は今日の日付から具体日に変換。
2. **スクリプトを実行する:**
   ```
   python {{REPO_PATH}}\scripts\mountain_weather.py --name 燕岳 --date 2026-07-19
   ```
   出力には常に「直近実況(過去3日)・16日間の見通し・開始日から4日分の詳細・
   JMA/ECMWF/GFSの3モデル比較」がすべて含まれる（「来週登れそうな日は?」も16日見通しで答えられる）。
   主な引数:
   - `--name 山名` … 内蔵DB(604座)→ジオコーディングの順で解決
   - `--select N` … 同名山が複数あると候補一覧が出て exit 2 になるので、ユーザーに確認して番号で再実行
   - `--lat --lon --elev --label` … DBにない山・任意地点の直接指定（--elev は山頂標高）
   - `--date YYYY-MM-DD` … 詳細表示の開始日（省略時は今日から）
   - `--interval 1` … 詳細を1時間ごとに表示（既定は3時間ごと。ピンポイントの時間確認に）
   - `--html [PATH] --open` … 色分きHTMLレポートを保存しブラウザで表示。
     「HTMLで」「見やすく」「保存して」等の依頼時に付ける（PATH省略時は自動命名）
3. **出力（Markdownテーブル）をそのまま提示し、解説を添える。** 解説で必ず触れること:
   - 行動判断に効く点: 稜線風が強まる/弱まる時間帯、降水・雷（CAPE）のタイミング、
     体感温度と装備（防寒・雨具）
   - 直近実況（冒頭の「直近の実況」表・過去3日のモデル解析値）から登山道コンディション
     （ぬかるみ・増水・新雪・残雪・アイスバーン）に触れる。積雪列「85cm(+12)」は
     積雪深max85cm・新雪12cmの意味（雪の無い期間は列ごと非表示）
   - 眺望: 眺望列（◎/○/△/✕、雲海付記）から展望が狙える時間帯・ご来光の可否
     （日の出時刻は詳細表ヘッダに表示）。「(雲海)」が付く朝は積極的に言及する
   - 夏山で午後にCAPEが上がる日は「早出早着・13時までに樹林帯へ」を明示
   - 対象日が4日以上先なら「まだ変わりうる。直前に再確認を」と添える
   - 判定基準の詳細は {{REPO_PATH}}\references\criteria.md を参照（風速・CAPE等の閾値の根拠）
4. **参考リンクを添える**（tozan-planの慣例に準拠）:
   - ヤマテン: https://i.yamatenki.co.jp/ ・ tenki.jp山の天気: https://tenki.jp/mountain/
5. **深い解説が求められたら** sangaku-kishou スキル（気象庁一次資料による総観場解説・GO/NG判断）を併用する。本スキル=数値地点予報、sangaku-kishou=総観場・気圧配置の解釈、という分担。

## エラー・特殊ケース

- **exit 2（候補複数）**: 候補一覧をユーザーに提示して選んでもらい `--select N` で再実行
- **16日超先の日付**: スクリプトがエラーを返す。「予報範囲外」の旨と16日見通しの範囲で答えられることを伝える
- **過去日**: 非対応（予報のみ）。求められたら気象庁の過去データ等を案内
- **DBにない山**: ジオコーディングにフォールバックする。それでも見つからない場合は
  WebSearch等で山頂の緯度経度・標高を調べて `--lat --lon --elev --label` で実行
- **ジオコーディング解決時の注意**: 標高はグリッド値で実際の山頂より低いことがある。
  正確な山頂標高が分かるなら `--elev` で上書きする

## 免責（出力に自動で含まれる）

数値予報は山岳では誤差が大きい。登山指数は目安であり、最終判断は最新予報と現地状況で行う。
スクリプト出力末尾の注意書きを削らずに伝えること。
